## 🔧 문제 해결

### 1. Supabase 연결 실패 시
- SUPABASE_ANON_KEY가 정확히 설정되었는지 확인
- 환경변수에 공백이나 특수문자가 없는지 확인

### 2. AI 기능 작동 안 함
- GOOGLE_AI_API_KEY가 유효한지 확인
- Google AI API 할당량 확인

### 3. 빌드 실패 시
- Vercel 함수 타임아웃 설정 확인 (30초)
- 메모리 사용량 확인 (1024MB)

## 📊 배포 후 확인사항

2. ✅ 대시보드 데이터 표시 확인  
3. ✅ AI 채팅 기능 테스트
4. ✅ 서버 모니터링 데이터 확인

## Vercel CLI 명령어로 환경변수 설정

```bash
# 필수 환경변수 설정
vercel env add SUPABASE_URL production
vercel env add SUPABASE_ANON_KEY production
vercel env add SUPABASE_SERVICE_ROLE_KEY production
vercel env add GOOGLE_AI_API_KEY production
vercel env add GOOGLE_AI_ENABLED production
vercel env add TARGET_SERVER_COUNT production
vercel env add DEFAULT_SERVER_COUNT production
vercel env add FALLBACK_SERVER_COUNT production
vercel env add FIXED_SERVER_COUNT production
vercel env add DISABLE_LOCAL_DATA_GENERATION production
vercel env add USE_GCP_FUNCTIONS_ONLY production
vercel env add NODE_ENV production
vercel env add VERCEL_DEPLOYMENT production
vercel env add DISABLE_LOCAL_RAG production
vercel env add DISABLE_HEALTH_CHECK production
vercel env add DISABLE_CRON_JOBS production
vercel env add AI_ENGINE_PRIORITY production
vercel env add BUILD_TARGET production
vercel env add SLACK_WEBHOOK_URL production
vercel env add ENABLE_BROWSER_NOTIFICATIONS production
vercel env add GCP_FUNCTIONS_BASE_URL production
vercel env add GCP_ENTERPRISE_METRICS_URL production
```
