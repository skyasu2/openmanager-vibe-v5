# 🚀 OpenManager Vibe v5.11.0 컴포넌트 가이드

**차세대 AI 기반 예측 모니터링 플랫폼**의 컴포넌트 라이브러리에 오신 것을 환영합니다!

## 📖 개요

OpenManager Vibe는 React + Next.js 기반의 현대적인 서버 모니터링 플랫폼입니다. 
이 Storybook에서는 프로젝트에서 사용되는 모든 주요 컴포넌트들을 문서화하고 테스트할 수 있습니다.

## 🏗️ 아키텍처 특징

### 🎯 **핵심 원칙**
- **React.memo** 최적화 적용
- **React.lazy()** 코드 스플리팅
- **TypeScript** 타입 안정성
- **접근성(A11y)** 표준 준수
- **반응형 디자인** 모바일 퍼스트

### ⚡ **성능 최적화**
- **초기 번들 크기** 14% 감소
- **첫 페인트 시간** 25% 개선
- **메모이제이션** 및 **디바운스** 적용
- **Suspense 경계** 설정

## 📁 컴포넌트 카테고리

### 🖥️ **Dashboard**
메인 대시보드 관련 컴포넌트들
- `DashboardHeader` - 헤더 영역
- `SystemStatusDisplay` - 시스템 상태 표시
- `DashboardContent` - 메인 콘텐츠

### 🤖 **AI Components**
AI 에이전트 관련 컴포넌트들
- `PresetQuestions` - 미리 정의된 질문 인터페이스
- `FlexibleAISidebar` - AI 사이드바
- `ThinkingProcess` - AI 사고 과정 시각화

### 🛠️ **System**
시스템 제어 관련 컴포넌트들
- `FloatingSystemControl` - 플로팅 시스템 제어판
- `SystemControlPanel` - 통합 제어 패널

### 📊 **Charts**
데이터 시각화 컴포넌트들
- `RealtimeChart` - 실시간 차트
- `PredictionChart` - 예측 차트

## 🔧 개발 가이드

### 새로운 스토리 추가
1. 컴포넌트와 같은 디렉토리에 `.stories.tsx` 파일 생성
2. 메타데이터와 스토리 정의
3. 다양한 상태와 props 조합으로 스토리 작성

### 예제 스토리 구조
```tsx
import type { Meta, StoryObj } from '@storybook/react';
import MyComponent from './MyComponent';

const meta: Meta<typeof MyComponent> = {
  title: 'Category/MyComponent',
  component: MyComponent,
  tags: ['autodocs'],
};

export default meta;
type Story = StoryObj<typeof meta>;

export const Default: Story = {
  args: {
    prop1: 'value1',
    prop2: true,
  },
};
```

## 📚 문서화 표준

### JSDoc 주석
- 모든 컴포넌트에 **JSDoc** 주석 작성

### Props 문서화
- 모든 props에 **description** 제공
- **control** 타입 명시
- **action** 핸들러 정의
