# 📋 Changelog - OpenManager V5

## [v5.21.0] - 2024-12-21 🚀 **AI 분석-오토스케일링 완전 분리**

### 🎯 **주요 기능: 구조적 분리 아키텍처**
- **AI 분석 로직과 서버 오토스케일링 완전 분리**
  - AI 분석: 고정된 8개 타겟 서버 기반 안정적 추론
  - 운영 시뮬: 8-30대 동적 서버 풀로 현실적 스케일링
  - 메타 이벤트: 스케일링 정보는 참조용으로만 제공

### ✨ **새로운 엔진 구조**
- **ScalingSimulationEngine**: 오토스케일링 전용 엔진
  - 동적 서버 풀 관리 (8-30대)
  - 실시간 스케일링 의사결정
  - 5분 쿨다운, 85% CPU 기준 scale-out
  - 30% CPU 기준 scale-in
  - 2서버씩 단위 스케일링
  
- **AdvancedSimulationEngine v4.0**: AI 분석 전용 엔진
  - 고정된 8개 AI 분석 대상 서버
  - primary(4), secondary(3), monitoring(1) 우선순위
  - 장애 시나리오는 분석 대상만 적용
  - 운영 서버 정보는 참조용 메타데이터로만 활용

### 🔧 **기술적 개선사항**
- **Redis 연결 안정화**
  - URL 파싱 오류 수정 (`ERR_INVALID_URL` 해결)
  - 더미 클라이언트 강화 (DummyRedisClient)
  - 개발/프로덕션 환경별 최적화
  - 타입 안전성 개선
  
- **데이터 일관성 보장**
  - AI 분석: realtime_metrics, daily_metrics 기반만
  - 스케일링: 운영 서버 상태만 참조
  - 메타 이벤트: 스케일링 기록은 로그용만

### 🧪 **새로운 API: `/api/simulate/ai-separation`**
- **`GET /?type=ai-only`**: AI 분석 대상만 조회 (고정 8개)
- **`GET /?type=scaling-only`**: 운영 서버 상태만 조회 (동적 8-30개)
- **`GET /?type=integrated`**: AI 통합 메트릭
- **`GET /?type=comparison`**: 분리 전후 비교 분석
- **`POST /`**: 스케일링 액션 실행
  - `trigger_scaling`: 강제 스케일링 시뮬레이션
  - `update_policy`: 스케일링 정책 업데이트
  - `start_ai_simulation`: AI 분석 시뮬레이션 시작
  - `stop_ai_simulation`: AI 분석 시뮬레이션 정지
- **`PUT /`**: 분리 테스트 실행
  - `scaling_stress_test`: 10회 스케일링 스트레스 테스트
  - `ai_consistency_test`: AI 분석 대상 일관성 검증

### 📊 **분리 효과 및 장점**
- **AI 추론 안정성**: 서버 수 변동과 무관한 일관된 분석 기준
- **운영 시뮬 유연성**: 8-30대 자유로운 스케일링으로 현실감 확보
- **비용 최적화**: AI는 최소 리소스(8개), 시뮬은 필요한 만큼
- **성능 테스트**: 대규모 환경 시뮬레이션과 AI 분석 병행
- **데이터 분리**: scaleEvent, newServer, retiredServer는 메타 정보로만 활용

### 🏗️ **아키텍처 설계**
```typescript
// 서버 생성기
const serverPool = autoScaler.simulate({ min: 8, max: 30 })
const metrics = generateMetrics(serverPool)

// AI 분석기 (완전 분리)
const activeTargets = filterByRole(fixedTargets, ['api', 'database']) 
const context = buildContextFrom(metrics, logs, alerts)
const report = analyze(context) // 고정된 서버 기반만
```

### 🔄 **호환성 및 마이그레이션**
- **기존 API 완전 호환**: `/api/simulate/advanced` 동일 동작
- **기존 AI 에이전트 무수정**: 코드 변경 불필요
- **점진적 전환**: 기존 시스템과 병행 운영 가능
- **테스트 격리**: 새 API로 분리 효과 검증 가능

### 📈 **Redis 및 Supabase 연동 최적화**
- **Redis**: 실시간 데이터 + 현재 서버 상태 저장
- **Supabase**: 과거 24시간 데이터 + 스케일 이벤트 기록 (AI용 참고 로그)
- **이중화**: Redis 실패 시 메모리 fallback 자동 전환
- **타입 안전**: TypeScript 기반 완전한 타입 체크

- **15% Offline**: 중단 상태 (CPU 0%, Memory 0%, 3-7개 심각한 경고)
