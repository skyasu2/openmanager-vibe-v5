### 2.2 메인 대시보드

**경로**: `/dashboard`

#### 레이아웃 구성
```
┌─────────────────────────────────────────────────────────┐
│ Header: 로고 + 전환 토글 + 사용자 메뉴                    │
├─────────────────────────────────────────────────────────┤
│ Statistics Cards (4개)                                   │
│ [전체 서버] [정상] [경고] [위험]                          │
├─────────────────────────────────────────────────────────┤
│ Server Grid (4x N)                                       │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐     │
│ │ Server 1 │ │ Server 2 │ │ Server 3 │ │ Server 4 │     │
│ └──────────┘ └──────────┘ └──────────┘ └──────────┘     │
│ ...                                                      │
├─────────────────────────────────────────────────────────┤
│ AI Assistant Button (우측 하단 플로팅)                    │
└─────────────────────────────────────────────────────────┘
```

#### 서버 카드 컴포넌트
- **표시 정보**: 서버명, 상태 뱃지, CPU/Memory/Disk 게이지
- **상태 색상**:
  - 정상 (healthy): 녹색 뱃지
  - 경고 (warning): 황색 뱃지
  - 위험 (critical): 적색 뱃지
- **인터랙션**: 클릭 시 상세 모달 오픈

#### 데모 데이터
- **총 서버 수**: 15대 (시뮬레이션)
- **실시간 업데이트**: ✅ 작동 확인

#### UX 평가
- **장점**: 직관적인 그리드 레이아웃, 실시간 메트릭 시각화
- **장점**: 상태별 색상 코딩으로 빠른 인지 가능
- **개선점**: 서버 필터링/검색 기능 미구현

---

### 2.3 서버 상세 모달

**트리거**: 서버 카드 클릭

#### 탭 구성

| 탭 | 내용 | 상태 |
|----|------|------|
| **종합 상황** | 실시간 메트릭 + 리소스 게이지 | ✅ 정상 |
| **성능 분석** | 차트 + 트렌드 분석 | ✅ 정상 |
| **로그 & 네트워크** | 로그 뷰어 + 네트워크 상태 | ✅ 정상 |

#### 종합 상황 탭
- CPU/Memory/Disk 사용률 프로그레스 바
- 현재 상태 뱃지 (정상/경고/위험)
- 최근 업데이트 타임스탬프

#### 성능 분석 탭
- 시계열 차트 (CPU/Memory 트렌드)
- 분석 기간 선택기
- AI 기반 패턴 분석 결과 표시 영역

#### 로그 & 네트워크 탭
- 로그 레벨 필터 (INFO/WARN/ERROR)
- 실시간 로그 스트림 뷰어
- 네트워크 레이턴시 표시

#### UX 평가
- **장점**: 잘 구조화된 3-탭 레이아웃
- **장점**: 각 탭이 명확한 역할 분리
- **개선점**: 탭 전환 시 애니메이션 부재

### 2.4 AI 어시스턴트 사이드바

**트리거**: 우측 하단 플로팅 버튼

#### 레이아웃
```
┌────────────────────────────┐
│ Header: AI 어시스턴트      │
│ [전체화면] [최소화]        │
├────────────────────────────┤
│ Feature Buttons (4개)      │
│ [자연어 질의] [자동장애]   │
│ [이상감지] [AI 상태관리]   │
├────────────────────────────┤
│ Chat Area                  │
│ - Message History          │
│ - Streaming Response       │
├────────────────────────────┤
│ Input Area                 │
│ [텍스트 입력] [전송]       │
└────────────────────────────┘
```

#### 기능별 상태

| 기능 | UI 상태 | 백엔드 연결 | 비고 |
|------|---------|-------------|------|
| 자연어 질의 | ✅ 정상 | 503 (Cloud Run) | 예상된 에러 |
| 자동장애 보고서 | ✅ 정상 | 500 (API Error) | Cloud Run 미연결 |
| 이상감지/예측 | ✅ 정상 | 미테스트 | 설정 UI 작동 |
| AI 상태관리 | ⏳ Coming Soon | N/A | 구현 예정 |

#### 자연어 질의 (Chat)
- **입력 방식**: 텍스트 + Ctrl+Enter 전송
- **응답 형식**: 스트리밍 (AI SDK v5 Data Stream)
- **에러 처리**: 503 에러 시 적절한 에러 메시지 표시

#### 자동장애 보고서
- **UI 요소**:
  - 보고서 목록 테이블
  - "새 보고서" 생성 버튼
  - 필터/검색 기능
- **에러 처리**: 500 에러 시 토스트 알림

#### 이상감지/예측
- **설정 옵션**:
  - 대상 서버 선택 드롭다운
  - 분석 깊이 (기본/상세)
  - 분석 단계 체크박스
- **분석 유형**: 이상 탐지, 트렌드 예측, 패턴 분석

#### AI 상태관리
- **현재 상태**: "Coming Soon" 플레이스홀더
- **예정 기능**: AI 모델 상태 모니터링

#### BUG-001 상세
- **재현 경로**: AI 사이드바 → "전체화면으로 보기" 클릭

## 7. 결론

### 7.1 전반적 평가

**OpenManager VIBE v5.83.7**은 모던한 다크 테마 UI와 직관적인 서버 모니터링 대시보드를 제공합니다. AI 어시스턴트 통합은 잘 설계되어 있으나, 일부 라우팅 이슈(`/ai` 404)가 수정 필요합니다.

### 7.2 강점
1. 깔끔하고 일관된 다크 테마 디자인
2. 직관적인 서버 상태 시각화 (색상 코딩)
3. 잘 구조화된 AI 사이드바 인터페이스
4. 실시간 메트릭 업데이트

### 7.3 개선 필요 영역
1. **BUG-001**: `/ai` 라우트 404 에러 수정 필수
2. AI 상태관리 기능 구현 (현재 Coming Soon)
3. 서버 필터링/검색 기능 추가
4. 접근성(A11y) 개선

### 7.4 다음 단계
1. BUG-001 수정 (High Priority)
2. 프로덕션 환경에서 Cloud Run 연동 테스트
3. E2E 테스트 시나리오 업데이트
