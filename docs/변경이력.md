- 모니터링 기능 초기 구조 설계

- 프로젝트 문서 한글화
- 시스템 구조 및 기술 스택 정의

- 코드 스타일 가이드 적용

## 커밋 컨벤션

본 프로젝트는 다음과 같은 커밋 메시지 컨벤션을 사용합니다:

- `feat:` 새로운 기능 추가
- `fix:` 버그 수정
- `docs:` 문서 변경
- `style:` 코드 포맷팅, 세미콜론 누락 등 (코드 변경 없음)
- `refactor:` 코드 리팩토링
- `perf:` 성능 개선
- `test:` 테스트 추가 또는 수정
- `chore:` 빌드 프로세스 또는 보조 도구 변경
