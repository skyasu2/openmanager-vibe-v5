## 🎯 바이브 코딩이란?

**바이브 코딩(Vibe Coding)**은 OpenManager Vibe v5 프로젝트에서 개발한 AI 기반 고효율 개발 방법론입니다.

### 핵심 철학

```markdown
🧠 AI와 개발자의 완벽한 협업
📚 컨텍스트 인식 기반 지능형 코딩
🔄 지속적 학습과 개선
⚡ 개발 속도와 품질의 동시 향상
```

### 바이브 코딩의 4대 원칙

1. **컨텍스트 우선주의**: 프로젝트 구조와 규칙을 AI가 완벽히 이해
2. **점진적 완성**: 작은 단위로 완벽하게 구현하며 확장
3. **실시간 피드백**: 코드 작성과 동시에 검증 및 개선
4. **지식 축적**: 프로젝트별 학습된 패턴을 지속적으로 활용

## 🛠️ Cursor AI 환경 설정

### 1. 필수 Cursor 설정 (실제 사용 경험 기반)

#### AI 기본 설정

```json
// Settings → Cursor → AI
{
  "cursor.ai.enableCodeActions": true,
  "cursor.ai.enableInlineCompletions": true,
  "editor.inlineSuggest.enabled": true,
  "editor.suggest.preview": true,
  "editor.acceptSuggestionOnEnter": "on"
}
```

#### 편의성 설정 (초보자가 놓치기 쉬운 것들)

```json
// Settings → General
{
  "editor.minimap.enabled": true,
  "editor.lineNumbers": "on",
  "editor.bracketPairColorization.enabled": true,
  "editor.guides.bracketPairs": true,
  "files.autoSave": "afterDelay",
  "files.autoSaveDelay": 1000,
  "editor.formatOnSave": true
}
```

### 2. 단축키 활용 (꼭 알아야 할 것들)

```markdown
Ctrl + K : AI Chat 열기 (가장 중요!)
Ctrl + L : 현재 파일을 AI에게 설명
Ctrl + I : 인라인 AI 편집
Tab : AI 제안 수락
Esc : AI 제안 거부
Ctrl + Shift + P : 명령 팔레트 (모든 기능 검색)
```

### 3. 초보자가 놓치기 쉬운 팁들

#### AI Chat 활용법

```markdown
✅ 좋은 질문: "이 React 컴포넌트에서 상태 관리를 위해 useState를 사용하고 있는데, 성능 최적화를 위해 useMemo를 추가해줘"
❌ 나쁜 질문: "코드 고쳐줘"

✅ 파일 첨부: Ctrl + L로 현재 파일 컨텍스트 제공
✅ 단계별 요청: 복잡한 작업을 작은 단위로 나누기
```

#### 프로젝트 컨텍스트 최적화

```markdown
- 관련 없는 파일들은 닫기 (AI가 혼동함)
- 현재 작업 중인 파일만 열어두기
- 에러 메시지나 로그는 정확히 복사해서 제공
```

### Project Rules (간단하고 실용적으로)

`Settings` → `Rules` → `Project Rules`:

```markdown
# OpenManager v5 프로젝트 규칙

- AI 엔진은 src/services/ai/ 디렉토리에 구성
- 로그에 이모지 사용: console.log('📊 데이터 로딩 완료')
- 파일명은 kebab-case: advanced-simulation-engine.ts
- 실제 데이터 우선, 더미 데이터는 폴백으로만 사용
```

### 💡 실제 개발에 도움되는 추가 팁

#### 1. Cursor에서 자주 사용하는 AI 명령어들

```markdown
"이 함수에 타입스크립트 타입 추가해줘"
"이 코드를 React Hook으로 리팩토링해줘"  
"에러 처리 로직 추가해줘"
"이 API 호출을 async/await으로 변경해줘"
"console.log 대신 적절한 로깅 추가해줘"
```

#### 2. 파일 네이밍 패턴 (실제 사용 중)

```markdown
컴포넌트: PascalCase (ServerCard.tsx)
훅: camelCase (useServerMetrics.ts)
유틸리티: camelCase (formatMetrics.ts)
서비스: kebab-case (advanced-simulation-engine.ts)
타입: PascalCase (ServerMetrics.ts)
```

#### 3. 개발 효율을 위한 간단한 규칙들

```markdown
- 함수는 한 가지 일만 하도록 작게 만들기
- async 함수에는 항상 에러 처리 추가
- 복잡한 로직에는 주석으로 설명 추가
- 테스트하기 쉬운 구조로 작성
```

## ⚡ 실전 워크플로우

### 1. 일일 개발 루틴

#### 시작 체크리스트 (5분)

```bash
npm run health-check      # 시스템 상태
npm run mcp:health        # MCP 연결
git pull origin main      # 최신 코드
npm run validate:quick    # 개발 환경 검증
```

#### 바이브 코딩 세션 (25분)

```markdown
1. **컨텍스트 설정** (2분): 작업 내용을 AI에게 명확히 설명
2. **집중 코딩** (20분): TDD + AI 실시간 협업
3. **검증 및 정리** (3분): 테스트 실행 + 품질 확인
```

### 2. 기능 개발 프로세스

#### Phase 1: 계획 및 설계

```bash
# AI와 요구사항 분석
curl -X POST localhost:3001/api/ai/analyze-requirements

# 설계 검증
npm run validate:design
```

#### Phase 2: 구현 (바이브 코딩)

```typescript
// 1. 인터페이스 정의 (AI 도움)
interface ServerMetrics {
  server_id: string;
  cpu: number;
  memory: number;
}

// 2. 테스트 작성 (AI 협업)
describe('MetricsEngine', () => {
  it('should process real data', async () => {
    // AI가 테스트 케이스 제안
  });
});

// 3. 구현 (AI 코드 생성)
// 4. 리팩토링 (AI 최적화)
```

#### Phase 3: 검증 및 배포

```bash
npm run validate:all      # 품질 검증
npm run test:performance  # 성능 테스트
npm run deploy-check      # 배포 준비
```
